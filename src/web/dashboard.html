Quick Start

🔍 Discover MCP Services

Go to the Guide tab to browse all registered MCP services. Use the search bar to find by name, or click AI for smart recommendations.

  • Filter by security level (S1-S4) and tags
  • Click a card to see details, capabilities, and install config

⚡ Install Services

Go to the Plugins tab and click + Add MCP Service to register and install a new MCP service.

  • Plugin — local TypeScript plugin (./plugins/xxx)
  • Proxy — remote MCP endpoint (http://host/mcp)
  • Standalone — external command (npx @some/mcp)

⚙ LLM Configuration

Click the gear icon (⚙) in the Guide toolbar to configure LLM providers for AI-powered search and recommendations.

  • Supports OpenAI-compatible, Anthropic, and Ollama
  • Test connection before saving

💻 Client Configuration

Connect to this MCP server from your AI assistant:

  • stdio: node dist/index.js
  • HTTP: http://localhost:3080/mcp
  • Click "Copy" in service details to get ready-to-paste config

Security Level Certification

🔒 Security Level System

Every MCP service has a security level (S1–S4) indicating its trust and audit status:

S1 OfficialBuilt-in or official trusted source, sandbox verified
S2 VerifiedThird-party verified, community reviewed and tested
S3 CommunityCommunity contributed, basic functionality checks passed
S4 UnverifiedNewly added, not yet reviewed — use at your own risk

📈 Upgrade Path

Services can apply for security level upgrades through a sandbox validation process:

  • S4 → S3: Basic check — has description, valid install, at least 1 tool
  • S3 → S2: Community review — source code available, tested by 3+ users
  • S2 → S1: Official audit — built-in source or trusted official repository, sandbox verified

☑ Auto-Check Criteria

Click "Run Sandbox" or "Upgrade with Sandbox" in a service's detail page. The system automatically verifies:

  • Has complete metadata (description, source, license)
  • Declares a semantic version such as 1.0.0
  • Declares at least one capability (tool/prompt/resource)
  • Install config is valid and complete
  • Known dangerous shell patterns are blocked
  • Proxy endpoint and source URL are checked with strict timeout when applicable

Plugin Development

🔨 Create a Plugin

Implement the McpPlugin interface and export as default:

import { z } from "zod";
import type { McpPlugin } from "aix-mcp-server/plugin";

const plugin: McpPlugin = {
  name: "my-plugin",
  register(server) {
    server.registerTool("my-tool", {
      title: "My Tool",
      description: "Does something",
      inputSchema: z.object({ input: z.string() }),
    }, async ({ input }) => ({
      content: [{ type: "text", text: input }],
    }));
  },
};
export default plugin;

See examples/mcp-plugin-example/ for a full template.
